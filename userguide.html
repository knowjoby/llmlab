🎨 Confidence-Coloured Text

Every character in the generated text gets a background colour showing how confident the model was when it chose that character. You can read the entire generation at a glance — no log needed.

The colour scale

dc <15% de 15–30% ar 30–50% e 50–65% st 65–80% ! >80%
ColourConfidenceWhat it means
Deep red < 15% Model was genuinely guessing — many characters were equally plausible
Orange 15 – 30% Low confidence — the model leaned this way but wasn't convinced
Yellow 30 – 60% Moderate confidence — a reasonable guess with real competition
Light green 60 – 80% Fairly confident — this was clearly the best option
Bright green > 80% Very confident — almost no doubt about the next character

What patterns to look for

  • Green streaks — the model is completing a familiar word or phrase it has seen many times in training
  • Red at the start of a line — the model is uncertain which character speaks next; many are plausible
  • Yellow/orange in the middle of a word — the model has narrowed things down but still has real alternatives
  • Hover over any character to see the exact confidence percentage in a tooltip
ℹ️ Compare temperatures. Generate the same prompt twice — once at 0.6, once at 1.4. The low-temperature run will be mostly green (confident, repetitive). The high-temperature run will have more red and orange (adventurous, varied).

Bottom panel — Confidence (%)

This is the mirror image. Where uncertainty is high, confidence is low. Watch how they oscillate: the model is uncertain at word starts, then gains confidence as it builds familiar letter sequences.

💡 The rhythm of writing. You'll notice peaks of uncertainty at the start of every new word or line, then a rapid drop as the model recognises a familiar pattern. This is exactly how a trained language model builds structure: uncertainty about what to say, then certainty about how to finish it.

🌿 Branch — The Road Not Taken

After generating text, the Branch tool lets you rewind the model to any character in the sequence, then generate a completely different continuation from that exact point. Same context, new random sampling — like replaying a decision and seeing a different outcome.

ℹ️ What "branching" means. Language models are probabilistic. At every step, multiple characters were plausible — the model just picked one randomly (weighted by probability). Branch lets you re-roll that dice and see what else could have happened.

How to use it

1
Run a generation first

Click ✨ Generate with any prompt. The Branch section below is greyed out until a generation has been stored.

2
Pick the branch point

Use the "Branch from character number" slider. Think of it as saying "rewind to just before character 10 was chosen, and try again."
Good values to try: early steps (3–10) for very different paths, later steps (50+) for small divergences.

If you choose a step beyond the length of the generated text, the app automatically clamps to the last available step and tells you in the Branch log: "you selected N — clamped to end of generated text".

3
Click 🌿 Branch from here

The model generates 120 new characters from that context. The branch output appears below with its own coloured text and confidence chart.

4
Compare original vs branch

The original generation is still visible above. Notice how the two paths diverge — the branch text starts from the same context but quickly becomes completely different. Same model, same training, different random moment.

💡 Best experiment. Branch from step 1 several times in a row. Each time you'll get a completely different Shakespeare-style passage from the same prompt. This directly demonstrates why language model outputs are never identical — and why temperature matters.

🔍 Verbatim Detector

After each generation, a collapsible accordion labelled "🔍 Verbatim check — is this copied or created?" appears inside the Generate tab, just below the confidence-coloured output. Click it to expand the Verbatim Detector, which answers one of the most interesting questions in AI: "Is the model creating something new, or just copying what it memorised?"

┌─ Verbatim check ──────────────────────────────────────────────────┐ │ │ │ To be, or not to be, that is the question │ │ │ │ Purple/blue = seen in training data │ │ Teal/cyan = novel — the model invented this │ │ │ │ 62% of this text appears in the original Shakespeare. │ └───────────────────────────────────────────────────────────────────┘

How it works

At startup, the app indexes every 6-character sequence from the 1.1 million-character training text. When you generate text, each 6-character window in the output is checked against this index. If it matches, those characters are highlighted in purple — the model is reproducing something it literally read during training. Teal characters are genuinely new combinations the model assembled itself.

ℹ️ Business analogy. Think of the training data as a library the model has read. Purple text is like quoting a book word-for-word. Teal text is the model's own synthesis — it learned the style and patterns, then applied them to something new. Most LLMs produce a mix of both.

What to look for

PatternWhat it means
High % purple (70–100%) The model is heavily reciting training text — useful to know when checking for memorisation
Low % purple (0–30%) The model is being more creative — especially at higher temperatures
Purple in names / stage directions Common Shakespeare phrases are well memorised — "ROMEO:", "Enter", "Exeunt" will almost always be purple
Teal in the middle of dialogue The model is interpolating — it learned structure but is generating new words
💡 Experiment. Generate the same prompt at temperature 0.3 (Focused) and again at 1.8 (Wild). The focused version will typically be more purple — the model plays it safe and sticks to memorised patterns. The wild version produces more teal.

Understanding Perplexity

Perplexity is a single number that summarises how surprised the model was by the whole text. Lower is better — it means the model predicted the text well. The app shows perplexity as "X out of max ~65" — because this model has a 65-character vocabulary, the worst possible perplexity is 65 (complete confusion across every character). Anything well below that means learning has happened.

Perplexity (out of max ~65)Plain English
1 – 3The model knows this text very well — almost memorised it
3 – 6The model has a reasonable grasp of it
6 – 12Parts of this are unfamiliar
12 – 30This text surprises the model a lot
30+The model finds this nearly incomprehensible
~65Maximum possible — equivalent to random guessing across all 65 characters

🌡️ Temperature Gallery

Temperature is the single most important setting in any AI language model. It controls how adventurous the model is. Low temperature = safe and predictable. High temperature = wild and creative. The Temperature Gallery lets you see this difference instantly by running the same prompt at five settings simultaneously.

┌─ Temperature Gallery ─────────────────────────────────────────────┐ │ │ │ Prompt: [To be, or not ] Length: [──●──] 80 │ │ │ │ [ 🔥 Compare all 5 temperatures ] │ │ │ │ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ 🌡️ 0.3 │ │ 🌡️ 0.6 │ │ 🌡️ 1.0 │ ... │ │ │ Focused │ │ Careful │ │ Balanced │ │ │ │ ▓▓▓▓ avg 82% │ │ ▓▓▓ avg 65% │ │ ▓▓ avg 47% │ │ │ │ To be, or │ │ To be, and │ │ To fell, or │ │ │ │ not to be, │ │ what if the │ │ shalt with │ │ │ └──────────────┘ └──────────────┘ └──────────────┘ │ └───────────────────────────────────────────────────────────────────┘

The five temperature settings

SettingLabelWhat happens
0.3 Focused The model almost always picks its top choice. Output is repetitive but grammatically safe.
0.6 Careful Some variety creeps in. The model occasionally takes a less-likely path.
1.0 Balanced The model's natural state. Probabilities are used as-is — the sweet spot for most use cases.
1.4 Creative Less-likely characters get a bigger chance. Outputs are more surprising and varied.
1.8 Wild Low-probability characters can now win. Outputs become erratic but occasionally poetic.
ℹ️ Business analogy. Temperature is like the confidence dial in a meeting. At 0.3, the model is a cautious consultant who always recommends the safest option. At 1.8, it's a brainstorming session where any idea — brilliant or bizarre — can win. Most production AI systems run somewhere between 0.6 and 1.2.
💡 Lower confidence does not mean worse writing. Each card in the gallery shows an average confidence badge. As temperature rises the badge drops — but that doesn't mean the output is bad. Lower confidence at higher temperatures means the model is exploring more options. Creative writing often benefits from that exploration.

Understanding the perplexity chart

The chart shows perplexity on the y-axis and training step on the x-axis. Perplexity measures how surprised the model is by standard Shakespeare text — lower means less surprised, which means it has learned more. Watch the dramatic drop in the first 200 steps — that's when the model goes from random to having learned basic letter frequencies.

ℹ️ What perplexity numbers mean here. A perplexity of 65 means the model is equally uncertain between all 65 characters at every step — like rolling a 65-sided die. At perplexity 5, it's down to 5 likely choices for the next character. A well-trained Shakespeare model reaches around 3–6. Watch the chart fall from ~65 toward that range as the replay progresses.
ℹ️ Business analogy. Imagine a new employee reading your company's entire document library. On day 1 (step 1), they produce word-salad. By week 1 (step 100), they're stringing sentences together. By month 1 (step 1000), they write convincingly in your company's style. The perplexity chart is their "how lost do I feel?" score over time.

🔭 Attention Heatmap

ℹ️ Why does attention matter? Before attention was invented, language models read text one character at a time and quickly forgot what came before. Attention lets the model "look back" and focus on whichever earlier characters are most relevant — like being able to re-read a sentence before finishing it. This is what makes transformers (and ChatGPT) dramatically better than older AI models. The heatmap below makes that invisible mechanism visible.

The Attention Heatmap reveals one of the most important mechanisms inside any modern AI model: attention. When the model decides what character comes next, it doesn't look at all previous characters equally — it focuses on certain ones more than others. This tab lets you see exactly where that focus lands.

Layer heatmaps (bottom grid) One panel per transformer layer. Rows = the character being predicted, columns = characters it can look back at. The heatmap uses a blue colour scale — darker/brighter blue = stronger attention connection.
Summary sentence Plain English: "When deciding the next character, the model focused most on 'X' (NN%) and 'Y' (NN%)."
ℹ️ Business analogy. Attention is like a manager reviewing a document. They don't read every word with equal care — they skim most of it but pay close attention to key words (the subject, the action, the deadline). The attention heatmap shows you which "key words" the model is tracking when it makes each decision.

How to use it

1
Load a model first

Go to the Train tab and click 📂 Load saved model.

2
Type a context string

Anything from 4–18 characters. Longer inputs use only the last 18 characters. Try To be, or not or a character name like ROMEO:\nO.

3
Click 🔭 Analyse attention

The strip and 4-panel heatmap appear within a second. Read the summary sentence first.

4
Explore layer differences

Notice how earlier layers (Layer 1–2) often show a diagonal pattern — each position looks at its immediate neighbours. Later layers (Layer 3–4) often have richer, longer-range connections — the model is integrating context from far away.

💡 Best experiment. Try the context JULIET:\n and compare to ROMEO:\n. The model has learned that different characters follow different names — watch how the attention patterns differ.

📐 Metrics in Plain English

Entropy (bits) — "How many options does the model think it has?"

Entropy measures how spread out the probability is across all 65 possible next characters.

ValueLabel in the appReal example
0 – 0.5almost certainAfter wor → d is almost inevitable
0.5 – 1.5very confidentAfter ROMEO: → newline is strongly expected
1.5 – 2.5fairly sureAfter a space → a few common letters compete
2.5 – 3.2a bit uncertainStart of a new word — many letters are plausible
3.2 – 4.0lots of optionsFirst character on a new line — any speaker could appear
4.0+hard to decideThe model has barely learned what comes here
Confidence (%) — "How sure was the model about its actual pick?"

Simply the probability the model gave to the character it actually chose.

  • 80–99% — near-certain: e.g. the second l in will
  • 30–80% — leaning that way but open to alternatives
  • 5–30% — low confidence: temperature or uncertainty drove the pick
Temperature — "How adventurous should the model be?"
SettingEffectOutput style
0.3 – 0.6Top token dominates stronglyRepetitive but coherent
0.7 – 0.9Balanced — realistic samplingFocused, varied
1.0Raw probabilities used as-isDefault — good creative range
1.2 – 1.5Lower-ranked tokens get more chancesCreative, occasionally wild
1.8 – 2.0Nearly uniform distributionMostly nonsense

🤖 AI Development Agents

Three specialist AI agents are built into this project and are currently live. They live in .claude/agents/ and are automatically activated by Claude Code when you describe a task. You never need to name them explicitly — just describe what you want and Claude routes to the right one. The agent definitions are kept up to date as the project evolves — the Architect agent in particular is responsible for keeping this guide in sync with app.py.

ℹ️ What is an agent? Think of each agent as a specialist consultant on your team. The Reviewer only audits — it has no ability to edit files. The Architect only plans — it never writes code. The Tester runs scripts and reports results. Each has a different skill level and different permissions, like a real team.

🔍 Reviewer

Model: Sonnet (fast, precise)
Permissions: Read-only — cannot edit any file
Job: Audits code for bugs, edge cases, and deviations from the project's known patterns. Knows every project-specific gotcha (hex colors, Gradio 6 API, checkpoint naming, etc.).

🏛️ Architect

Model: Opus (slowest, deepest reasoning)
Permissions: Read-only — produces plans only
Job: Designs new features before any code is written. Knows the 6-tab structure, the global state, and the non-technical user persona. Always delivers a written plan with plain-English descriptions.

🧪 Tester

Model: Haiku (cheapest, fastest)
Permissions: Can read files, write temp test scripts, and run terminal commands
Job: Writes a minimal test, runs it with .venv/bin/python, reports pass/fail for each function, then cleans up. Never edits the main app.

How to trigger each agent

Just describe your task naturally. Claude Code reads the request and routes to the right agent automatically.
